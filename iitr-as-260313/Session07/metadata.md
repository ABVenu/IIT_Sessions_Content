course name: Certification in Agentic systems and design

target audience: students of any background - not from tech bakground

session duration: 2 hr 15 mins

title: Defining Agentic Systems & Core Components	

objective: Understand the fundamental components that make an AI system agentic	

topics to be covere: planning; reasoning; memory; action	

detailed subtopics/learning objectives to be covered: 
* Define Agentic Systems — Understand what an AI agent is, how it differs from traditional systems, and identify simple real-world examples;
* Identify Agent Characteristics — Recognize autonomy, goal-driven behavior, and ability to take actions in simple terms;
* Break Down a Goal into Steps — Learn how a task is split into smaller steps using everyday examples (e.g., booking a ticket);
* Understand Planning Basics — See how agents decide “what to do next” without going into algorithms;
* Understand Simple Decision Making — Learn how agents choose between options based on available information;
* Understand Memory Concept — Learn why remembering past interactions is useful (like chat history);
* Differentiate Memory Types — Distinguish short-term (current task) vs long-term (stored info) with simple analogies;
* Understand Actions in Agents — Learn what actions mean (API call, tool usage, or simple task execution);
* Follow the Agent Flow — Understand Observe → Think → Act in a simplified manner;
* Connect All Components — See how planning, reasoning, memory, and action work together in one flow;
* Analyze a Simple Use Case — Break down a relatable task (e.g., food ordering assistant) into agent components;
* Identify Common Mistakes — Understand where agents can go wrong (wrong step, forgetting context, wrong action);
