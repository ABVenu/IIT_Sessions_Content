# Pre-Read: Setting Up Your Lab

## What You’ll Learn
In this pre-read, you’ll discover:
- Why having a proper **development lab** matters before writing any AI code  
- What **VS Code** and **Google Colab** are used for (and when to use each)  
- What **API keys and secrets** are, and why they must be protected  
- Why a **GitHub account** is essential for modern developers  

---

## Introduction: What Is a “Lab” in AI?
Think about learning to cook.

You wouldn’t start without:
- A kitchen  
- Basic utensils  
- Ingredients stored properly  

An AI **lab** is the same idea—but for coding.

Your lab is the **environment** where you:
- Write code  
- Run experiments  
- Store projects  
- Collaborate with others  

A good setup saves time, avoids errors, and makes you feel confident as a developer.

---

## Why Setting Up a Lab Matters
Many beginners jump straight into coding and struggle later.  
A proper setup solves real problems:

1. **Consistency**  
   Your code behaves the same every time you run it.

2. **Productivity**  
   Tools like editors and notebooks speed up development.

3. **Security**  
   Sensitive keys stay private and safe.

4. **Professional habits**  
   You work the way real teams and companies do.

---

## From Known to New: Files → Development Environment
### The Simple Way (What You Already Know)
You may have written code in:
- A basic text editor  
- An online compiler  
- A single file saved locally  

This works for small experiments—but breaks down quickly.

### The Professional Way
You use:
- A **code editor** (VS Code)  
- A **cloud notebook** (Colab)  
- **Version control** (GitHub)  
- **Secure configuration** (API keys)

This lecture shows how these pieces fit together.

---

## Core Components of Your AI Lab
### 1. Visual Studio Code (VS Code)
A **code editor** where you write and manage files.

- Runs on your computer  
- Supports extensions, debugging, and formatting  
- Feels like a professional workspace  

*Think of it as your main desk.*

---

### 2. Google Colab
A **cloud-based notebook** that runs code online.

- No installation needed  
- Free access to GPUs  
- Ideal for experiments and learning  

*Think of it as a shared lab bench.*

---

### 3. Secrets and API Keys
Some tools require **keys** to access services.

- These keys act like passwords  
- Sharing them publicly is dangerous  
- They must be stored securely  

*Never hard-code secrets into files you share.*

---

### 4. GitHub
A platform to **store, track, and share code**.

- Saves versions of your work  
- Makes collaboration possible  
- Acts as your public developer portfolio  

*Think of it as your project archive + resume.*

---

## How These Pieces Work Together
A typical flow looks like this:
1. Write and organize code in **VS Code**  
2. Experiment quickly in **Colab**  
3. Store secrets safely using environment tools  
4. Push clean, professional code to **GitHub**

Each tool has a role. None is optional in real-world work.
