Course Name: Certification in Agentic Systems and Design

Target Audience : Students from any backgorund may not be necessarily form tech background


session duration: 2hr 15 mins

title: Implementing Vector Search Systems

objective: Understand how to store and query embeddings in vector databases

topics be covered: storing embeddings;querying vector DB


detailed subtopics to be covered:
- Define the Objective of a Vector Search System — Establish the goal of building a system that retrieves semantically similar results using embeddings;
- Understand the End-to-End Implementation Workflow — Outline the pipeline: data → embeddings → storage → query → retrieval;
- Set Up the Development Environment — Install and configure a beginner-friendly vector database such as Chroma along with required libraries;
- Prepare Sample Data for Implementation — Structure a small dataset (documents/sentences) for embedding and retrieval;
- Generate Embeddings Using Pretrained Models — Convert text data into embeddings using an API or local library;
- Initialize a Vector Database Collection — Create a collection (or index) to organize and store embeddings;
- Store Embeddings in the Vector Database — Insert vectors along with identifiers and metadata into the collection;
- Verify Stored Data — Perform basic checks to ensure embeddings are successfully stored and accessible;
- Convert Query Input into Embedding — Transform user queries into embeddings for similarity search;
- Execute Similarity Search Queries — Retrieve top-k most similar results from the vector database;
- Interpret Query Results — Understand returned results, similarity scores, and ranking;
- Implement Top-K Retrieval — Configure retrieval to return a fixed number of most relevant results;
- Apply Basic Metadata Filtering in Queries — Combine similarity search with simple filters (e.g., category, tags) to refine results;
- Update the Vector Collection — Add new data and embeddings to an existing collection;
- Build a Minimal End-to-End Vector Search Application — Integrate all steps into a working pipeline from input query to retrieved results;
